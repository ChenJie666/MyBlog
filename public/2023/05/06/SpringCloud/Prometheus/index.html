#一、简介
监控一般分为：业务级别监控/系统级别监控/网络监控/程序代码监控/日志监控/用户行为监控/其他种类监控。

  • 业务监控：包含用户访问QPS，请求响应延迟，DAU日活，访问状态（http code），业务接口（登录，注册，聊天，上传，留言，短信，搜索），产品转化率，充值额度，用户投诉等；
  • 系统监控：操作系统相关的基本监控项 CPU / 内存 / 硬盘 / IO / TCP链接 / 流量 等；
  • 网络监控：对网络状态的监控（交换机，路由器，防火墙，VPN），例如 丢包率，延迟等；
  • 日志监控：监控中的重头戏，需要单独设计和搭建，全部种类的日志都需要采集；
  • 程序监控：配合开发，在程序中嵌入各种接口，直接获取数据或特制的日志格式。

Prometheus主要是针对系统监控。

exporter包括：

  • alertmanager
  • blackbox_exporter
  • consul_exporter
  • graphite_exporter
  • haproxy_exporter
  • memcached_exporter
  • mysqld_exporter
  • node_exporter
  • pushgateway (可以通过脚本抓取数据然后主动推送到prometheus服务器，更灵活便捷)
  • statsd_exporter

#二、安装框架 **准备三台服务器，需要将服务器的时间进行同步** * * * * * ntpdate cn.ntp.org.cn

##2.1 安装prometheus
docker run -d –name prometheus –net host –restart=always -v /root/prometheus/config:/etc/prometheus -v “/etc/localtime:/etc/localtime” prom/prometheus

默认的配置文件

可以添加额外的监控节点

访问地址
http://192.168.32.244:9090

##2.2 安装node-exporter
作用：node-exporter用于后台采集服务器的数据，以守护进行运行在后台不断进行数据采集。
安装：docker run -d –net=host –pid=host –restart=always –name node_exporter -v “/:/host:ro,rslave” prom/node-exporter –path.rootfs=/host –collector.filesystem.ignored-mount-points “^/(sys|proc|dev|host|etc)($|/)”

启动后可以通过http://192.168.32.242:9100/metrics
获取节点的信息，配置到prometheus.yml中进行监控。

##2.3 安装mysqld-exporter
作用：mysqld-exporter用于后台采集mysql的数据，以守护进行运行在后台不断进行数据采集。
docker run -d –net=host –restart=always –name=mysqlId -e DATA_SOURCE_NAME=”exporter:123456@(192.168.32.244:3306)/“ prom/mysqld-exporter

为prometheus创建用户并赋权限

启动后可以通过http://192.168.32.244:9104/metrics
获取节点的信息，配置到prometheus.yml中进行监控。

##2.4 安装Grafana
docker run -d –net=host –user=root –name=grafana -v /root/grafana/config/grafana.ini:/usr/share/grafana/conf/defaults.ini -v /root/grafana/data/:/var/lib/grafana/ grafana/grafana

配置datasource为已经部署的prometheus，添加图形，通过查询语句查询需要显示的数据。
